### **Problem: The Dynamic Matrix Class**

**Concepts Tested:** Pointers (specifically double pointers ``), Dynamic Memory Allocation (`new` and `delete`), Object-Oriented Programming (Constructors & Destructors), and Exception Handling.

**Scenario:** You are building a lightweight mathematics library. You need a `Matrix` class to represent a 2D grid of numbers. Because the dimensions of the matrix won't be known until the program is running, you cannot use standard static arrays. You must build it dynamically on the heap.

**Requirements:**

1. **The Class & State:** Create a class named `Matrix`. It should have three private member variables:
* `int rows`
* `int cols`
* `int data` (A pointer to a pointer, which you will use to create an array of arrays).


2. **Constructor:** * Create a constructor `Matrix(int r, int c)`.
* Inside the constructor, dynamically allocate memory for an `r` by `c` 2D array using the `data` pointer.
* Initialize all elements in the matrix to `0`.


3. **Destructor (Crucial Step):**
* Create a destructor `~Matrix()`.
* You must write the logic to safely `delete` the dynamically allocated memory. Remember: for every `new[]` you used in the constructor, you need a matching `delete[]` in the destructor to prevent memory leaks!


4. **Member Functions:**
* `void setValue(int r, int c, int value)`: Sets the element at row `r` and column `c` to the given `value`.
* `int getValue(int r, int c) const`: Returns the value at the specified row and column.
* **Boundary Checking:** For both of the above functions, check if `r` and `c` are valid indices (e.g., `r` must be $\ge 0$ and `< rows`). If they are out of bounds, `throw` a `std::out_of_range` exception.
* `void display() const`: Uses nested loops to print the matrix to the console in a clean grid format.


5. **The `main()` Function Implementation:**
* Instantiate a `Matrix` of size 3x3.
* Use a nested loop to populate it. (For example, set the value at `(i, j)` to `i * j`).
* Call `display()` to show the populated matrix.
* Intentionally attempt to call `setValue()` at an invalid index (like row 5, col 0) inside a **`try-catch`** block to test your error handling. Print the error message when caught.



**Expected Console Output (Example):**

--- Matrix (3x3) ---
0 0 0 
0 1 2 
0 2 4 

--- Testing Bounds ---
Exception caught: Index out of bounds!
