Problem: The Generic Sensor Data Tracker
Concepts Tested: Object-Oriented Programming, Templates, Introduction to STL (Vectors), Pairs, Exception Handling, References, and Loops.

Scenario: You are building a software component for an engineering system that logs sequential data from various types of sensors (e.g., temperature in decimals, or object counters in whole numbers). You need a reusable, safe way to store and analyze this data.

Requirements:

Template Class: Create a template class named SensorTracker<T> so it can handle different numeric data types (like int, float, or double).

Data Storage (STL & Pairs): Inside the class, use a std::vector to store a collection of std::pair<std::string, T>.

The std::string will represent a timestamp or reading ID.

The T will hold the actual sensor value.

Member Functions:
Implement the following methods:

void addReading(const std::string& timestamp, T value): Adds a new timestamp and value pair to your vector. (Hint: Use references to avoid copying the string).

void displayReadings(): Uses a loop to iterate through the stored vector and prints each timestamp and corresponding value.

T calculateAverage(): Computes and returns the average of all sensor values currently in the vector.

Exception Handling: In your calculateAverage() function, if the vector is empty, dividing by zero would cause an error. Instead, throw an exception (you can use std::runtime_error or design a custom exception class) indicating that there is no data to average.

The main() Function Implementation:

Instantiate a SensorTracker<double> to act as a temperature logger.

Add at least three sample readings to it (e.g., "08:00" -> 22.5, "09:00" -> 23.1, "10:00" -> 24.0).

Call displayReadings() and print the result of calculateAverage().

Instantiate a second, completely empty SensorTracker<int>.

Attempt to call calculateAverage() on this empty tracker inside a try-catch block to catch the exception and print a safe error message to the console.

Expected Console Output:

--- Temperature Readings ---
[08:00]: 22.5
[09:00]: 23.1
[10:00]: 24
Average Temperature: 23.2

--- Testing Empty Tracker ---
Error caught: Cannot calculate average. No sensor data available.
