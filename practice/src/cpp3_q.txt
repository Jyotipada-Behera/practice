### **Problem: The Template Data Chain**

**Concepts Tested:** Pointers to Objects, Dynamic Memory (`new`/`delete`), Templates, Object-Oriented Programming, and Exception Handling.

**Scenario:** You need to store a sequence of data, but you want to avoid the overhead of copying and resizing that comes with standard arrays or vectors when adding items to the front of the list. You will build a Singly Linked List template class.

**Requirements:**

1. **The Node Structure:**
* Create a template `struct` or `class` named `Node<T>`.
* It should contain two public members: the actual `data` (of type `T`), and a `next` pointer (a pointer to the next `Node<T>`).
* Include a constructor to easily initialize a node with a value and set `next` to `nullptr`.


2. **The List Class:**
* Create a template class named `LinkedList<T>`.
* It needs one private member variable: a `Node<T>* head` pointer, which keeps track of the start of the list.


3. **Core Methods:**
* **Constructor:** Initialize `head` to `nullptr` (an empty list).
* **Destructor (`~LinkedList`):** This is critical. You must loop through the entire list and `delete` every single node to prevent memory leaks.
* **`void push_front(T value)`:** Dynamically allocate a new node with the given value, and insert it at the very *beginning* of the list.
* **`void pop_front()`:** Remove the first node in the list and safely `delete` it. If the list is already empty, `throw` a `std::underflow_error` (from `<stdexcept>`).
* **`void display() const`:** Iterate through the nodes via their `next` pointers and print the data in a format like `[10] -> [20] -> [30] -> nullptr`.


4. **The Challenge Function (Pointer Gymnastics):**
* **`void reverse()`:** Reverse the entire linked list *in place*. You cannot allocate new nodes or use an array/vector. You must do this purely by rearranging the `next` pointers of the existing nodes. *(Hint: You will need to keep track of previous, current, and next nodes).*


5. **The `main()` Function:**
* Instantiate a `LinkedList<int>`.
* Push three numbers onto the front (e.g., `10`, then `20`, then `30`).
* Display the list (Note: because you push to the front, `30` should print first).
* Call `reverse()` and display the list again.
* Call `pop_front()` until the list is empty, plus one more time inside a `try-catch` block to catch and print your underflow exception.



**Expected Console Output (Example):**

```text
--- Initial List ---
[30] -> [20] -> [10] -> nullptr

--- Reversed List ---
[10] -> [20] -> [30] -> nullptr

--- Testing Pop ---
Popped all items.
Exception caught: List is empty!
